## 背景

我的工作位于研究与工程的交叉地带。
我关注 AI、Agent 系统、科学计算、电路设计、量子计算和 quantum memory 等方向，尤其在意如何把这些想法真正落到可用的工具、仿真流程和真实技术系统中。

## 代表项目

- [actoviq-agent-sdk](https://github.com/DeconBear/actoviq-agent-sdk)\
  一个独立开发、带有实验性质的 agent SDK，面向真实的多工具、多会话、bridge-assisted 工作流。这个项目关注的不是单纯展示 agent 能做什么，而是把 agent 系统做成真正可用的开发基础设施，包括 session、tools、MCP 集成、workspace helpers、runtime bridge，以及交互式流式工作流等能力。它比较集中地体现了我在 agent runtime 设计、系统编排和开发者体验上的兴趣。
- [SmithTool](https://github.com/DeconBear/SmithTool)\
  一个基于 C++17 与 Qt 6 Widgets 开发的交互式 Smith 圆图桌面应用，用于 RF 阻抗匹配与匹配网络设计。它支持阻抗编辑、匹配元件设计、Touchstone 文件导入、SPICE 网表导出，以及中英文双语界面。这个项目与我在电路设计、工程软件以及面向真实技术问题的工具开发上的兴趣密切相关。
- [klynx\_skills](https://github.com/DeconBear/klynx_skills)\
  一个面向工程工作流复用的 skills 仓库，目前重点放在结合 ngspice 的电路设计与仿真。它提供了从网表校验、批量仿真执行、指标提取，到报告导出和原理图渲染的一整套 simulation-feedback loop。这个仓库体现了我希望把 AI 辅助工作流、电路分析和计算型设计工具结合起来的方向。

## 参与项目

- [QuTiP](https://github.com/qutip/qutip)\
  一个用于量子系统模拟的开源框架，在量子光学、量子信息以及相关计算物理研究中被广泛使用。
- [QuTiP-qip](https://github.com/qutip/qutip-qip)\
  QuTiP 生态中的量子信息处理项目，聚焦量子线路层面的模拟与相关工作流支持。
